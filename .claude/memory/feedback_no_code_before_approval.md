name: no-code-before-approval

metadata:
  type: feedback
---

Never write code or create files until the user has explicitly approved the design or plan.

**Why:** User was shown a design and I started implementing immediately without waiting for their go-ahead. This wastes effort if the design needs changes and violates the user's workflow.

**How to apply:** When a design involves multiple files or architectural decisions, present the plan as text first and wait for explicit approval ("looks good", "go ahead", "approved", etc.) before touching any file.
